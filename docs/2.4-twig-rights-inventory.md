## 2.4 Twig templates / frontend – rights usage inventory

This document corresponds to plan section **2.4** and todos:

- `inventory-twig-navigation`
- `inventory-twig-feature-templates`

It lists Twig templates that reference `workstation.useraccount.rights.*`
or `rights.*` and briefly categorises them.

### Navigation / global layout (5.2)

- `zmsadmin/templates/block/navigation/navigation.twig`
  - Uses `rights.ticketprinter`, `rights.scope`, `rights.department`,
    `rights.organisation`, `rights.availability`, `rights.useraccount`.
  - Multiple checks for `workstation.useraccount.rights.superuser`
    (including string comparison `== "1"`).
- `zmsadmin/templates/block/page/configinfo.twig`
  - Checks `workstation.useraccount.rights.superuser == "1"`.
- `zmsadmin/templates/page/configinfo.twig`
  - Similar `workstation.useraccount.rights.superuser == "1"` check.
- `zmsadmin/templates/page/status.twig`
  - Uses `rights.superuser` to gate status information.

These templates will migrate to `permissions.*` and/or role-based checks
for menu visibility and global admin sections.

### Useraccount management UI (5.3 / 6.x)

- `zmsadmin/templates/block/useraccount/list.twig`
  - Top-level gate: `workstation.useraccount.rights.superuser`.
  - Per-row rights flags: `rights.audit`, `rights.scope`, `rights.cluster`,
    `rights.superuser`.
- `zmsadmin/templates/block/useraccount/listByDepartment.twig`
  - Uses `rights.audit`, `rights.scope`, `rights.cluster`, `rights.superuser`.
- `zmsadmin/templates/block/useraccount/departmentlist.twig`
  - Several `workstation.useraccount.rights.*` checks mapping to legacy
    numeric “role” levels (90, 70, 50, 40, 30, 5).
- `zmsadmin/templates/block/useraccount/edit.twig`
  - Uses `workstation.useraccount.rights.superuser`, `rights.scope`,
    `rights.cluster`, etc., to control editable fields and options.
- `zmsadmin/templates/block/useraccount/info.twig`
  - Uses `useraccount.rights.*` for explaining legacy roles to the user
    (superuser, organisation, department, cluster, useraccount, scope,
    availability, ticketprinter, sms, basic).
- `zmsadmin/templates/block/useraccount/profile/changepassword.twig`
  - Uses `useraccount.rights.superuser` to control visibility of options.

These templates will be migrated to reflect explicit roles and permissions,
and their explanatory texts updated per section 5.4 and 6.x.

### Organisation / owner / tree views (5.3)

- `zmsadmin/templates/block/owner/overview.twig`
  - Uses `rights.superuser`, `rights.department`, `rights.cluster`,
    `rights.scope`, `rights.availability` in various nested conditions.
- `zmsadmin/templates/block/organisation/form.twig`
  - Uses `workstation.useraccount.rights.organisation`,
    `workstation.useraccount.rights.department` to gate buttons and actions.

### Scope / cluster / calendar / appointment views (5.3)

- `zmsadmin/templates/block/scope/form.twig`
  - Multiple checks for `rights.superuser` controlling advanced scope
    configuration sections.
- `zmsadmin/templates/block/cluster/form.twig`
  - Uses `rights.superuser` for cluster-level operations.
- `zmsadmin/templates/block/cluster/scopeSelect.twig`
  - Contains commented-out check against
    `workstation.useraccount.rights.superuser`.
- `zmsadmin/templates/block/calendar/calendarMonth.twig`
  - Uses `workstation.scope` and `workstation.useraccount.rights.scope` to
    gate access to the calendar view.
- `zmsadmin/templates/block/appointment/times.twig`
  - Uses `workstation.scope` and `workstation.useraccount.rights.scope`.

### Notifications / status / misc (5.3 / 5.4)

- `zmsadmin/templates/block/notification/form.twig`
  - Uses `rights.sms` flag for SMS-related configuration.
- `zmsadmin/templates/block/useraccount/departmentlist.twig`
  - Also part of the useraccount management UI but important for role-level
    overview; uses `workstation.useraccount.rights.*` plus per-row `rights.*`.
- `zmsadmin/templates/page/status.twig`
  - `rights.superuser` for status blocks.

### Statistic navigation (5.3)

- `zmsstatistic/templates/block/statistic/departmentlist.twig`
  - Uses `workstation.useraccount.rights.superuser` in the statistics
    navigation area.

These usages will be migrated to `permissions.statistic` and/or appropriate
role checks (`reporting_viewer`, `appointment_admin`, etc.) per ticket 1172.
