## 2.2 DB and query layer – inventory

This document corresponds to plan section **2.2** and todos
`inventory-db-schema` and `inventory-useraccount-query`.

It records where `permission`, `role`, `role_permission`, `user_role`, and
`Berechtigung` are defined and used in the schema and query layer.

### 1. Schema and migrations for permissions/roles

- `zmsdb/migrations/91771408741-create-permissions-table.sql`
  - Defines `permission` table with `name` and `description`.
  - Inserts the frozen list of atomic permissions.
- `zmsdb/migrations/91771408752-create-roles-table.sql`
  - Defines `role` table (not re-listed here, see section 1.1 of the plan).
- `zmsdb/migrations/91771408763-create-role-permission-table.sql`
  - Defines `role_permission` table and populates bundles for each role.
- `zmsdb/migrations/91771408773-create-user-role-table.sql`
  - Defines `user_role` bridge table linking `user` ↔ `role`.
- `zmsdb/migrations/91773136364-alter-user-role-fk.sql`
  - Adjusts foreign-key semantics for `user_role.user_id` (cascade).
- `zmsdb/migrations/91771576480-migrate-users-to-new-roles.sql`
  - One-off migration mapping `Berechtigung` values to initial roles.

These migrations together define the target permissions/roles model and the
bridge from legacy numeric rights.

### 2. `Useraccount` query expressions based on `Berechtigung`

File: `zmsdb/src/Zmsdb/Query/Useraccount.php`

Computed `rights__*` fields (legacy, derived from `useraccount.Berechtigung`):

- `rights__superuser` → ``useraccount`.`Berechtigung` = 90`
- `rights__organisation` → ``useraccount`.`Berechtigung` >= 70`
- `rights__department` → ``useraccount`.`Berechtigung` >= 50`
- `rights__cluster` → ``useraccount`.`Berechtigung` >= 40`
- `rights__useraccount` → ``useraccount`.`Berechtigung` >= 40`
- `rights__scope` → ``useraccount`.`Berechtigung` >= 30`
- `rights__departmentStats` → ``useraccount`.`Berechtigung` >= 25`
- `rights__availability` → ``useraccount`.`Berechtigung` >= 20`
- `rights__ticketprinter` → ``useraccount`.`Berechtigung` >= 15`
- `rights__sms` → ``useraccount`.`Berechtigung` >= 10`
- `rights__audit` → ``useraccount`.`Berechtigung` = 5 OR `useraccount`.`Berechtigung` = 90`
- `rights__basic` → ``useraccount`.`Berechtigung` >= 0`

Direct `Berechtigung` filters:

- Equality filter: `where('useraccount.Berechtigung', '=', $roleLevel)`.
- Superuser exclusion filters (multiple places):
  - `where('useraccount.Berechtigung', '!=', 90)`.

These expressions and filters must eventually be replaced with role/permission
based logic (`user_role` + `role_permission`) as part of the migration.
